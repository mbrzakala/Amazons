# Agents

## General Rules

- Before starting any task, look for a corresponding `.md` file in the `agent_doc/` folder that covers the relevant domain or feature. Read it fully before proceeding.
- Every task requires sufficient context. If the task description is vague, incomplete, or missing key details, **ask clarifying questions before writing any code**.
- Do not make assumptions to fill in missing requirements — always request the missing information from the user.

## Context Resolution

1. Identify the task domain (e.g., game logic, UI, AI, API).
2. Locate the matching doc file: `agent_doc/<domain>.md`.
3. If no matching doc exists, ask the user where to find context or what the requirements are.
4. Only proceed once the task and its constraints are clearly understood.
