# Validator — 3. Problem Statement

Checks: `Hackathon Wiki/stages/1-discovery/steps/3-problem-statement.md`.

## What this step is proving

The persona's situation has been compressed into one testable sentence naming a need,
not a solution — with the four supporting canvas fields filled in behind it.

## Evidence to gather

- The four canvas fields: who (persona), what they need, why (root cause/pain), what
  exists today and why it falls short.
- The final one-sentence problem statement.

## Gate checklist

1. **All four canvas fields are filled in with specifics.**
   Evidence: the canvas. Fail if any field is empty or generic.
2. **The problem statement follows the exact form:** "[Persona] needs a way to [need],
   because [pain point]."
   Evidence: the sentence text. Fail if the structure doesn't match.
3. **The statement names a need, not a solution.**
   Evidence: sentence text. Fail if it contains solution language ("an app that," "an AI
   that," or equivalent — naming the mechanism instead of the need).
4. **Someone unfamiliar with the idea could judge pass/fail against this sentence
   alone.**
   Evidence: read the sentence in isolation, without any other context. Fail if it
   requires outside knowledge to parse.
