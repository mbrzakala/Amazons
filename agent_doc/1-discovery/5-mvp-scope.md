# Validator — 5. MVP Scope

Checks: `Hackathon Wiki/stages/1-discovery/steps/5-mvp-scope.md`.

## What this step is proving

Every step from the process map has an explicit MVP/Later decision, and the resulting
MVP is genuinely the smallest thing that proves the core value — not everything the team
thought was nice to have.

## Evidence to gather

- The step-4 process map/event list.
- The labeled (MVP/Later) version of it, and the written MVP scope list.

## Gate checklist

1. **Every step from step 4 is labeled MVP or Later.**
   Evidence: compare the full step-4 event list against the labeled version. Fail if any
   event is unlabeled.
2. **The MVP is the smallest set of steps that proves the core value — nothing extra.**
   Evidence: review each MVP-labeled item and ask whether removing it would still let
   the core value be demonstrated. Fail if items are included that aren't essential to
   proving the core value (this is a judgment call — flag borderline items rather than
   silently passing them).

See `agent_doc/README.md` for the verdict scale and reporting format.
