# Validator — 6. AI Agent Component Generation

Checks: `Hackathon Wiki/stages/2-design-system/steps/6-ai-agent-component-generation.md`.

## What this step is proving

The first real UI components were generated by prompting the MCP-connected design
system by name, use token names rather than hardcoded values, cover every documented
state/variant, and were visually verified to actually render.

## Evidence to gather

- Generation prompts/commit messages, checked for references to specific
  tokens/variants/states vs. vague styling language.
- Component source files, grepped for hardcoded hex/px values vs. `var(--token-name)`.
- The step-4 documented anatomy for the component(s) generated.
- A screenshot or live-render proof.

## Gate checklist

1. **Every generation prompt references the MCP-connected design system by name — no
   vague/freeform styling prompts.**
   Evidence: prompt/commit history. Fail on prompts like "make it look modern" with no
   token/variant reference.
2. **Generated code uses token names, not hardcoded hex/px values.**
   Evidence: grep component styles for raw hex/px vs. `var(--...)`.
3. **Every documented state and variant for that component is implemented, not just the
   happy-path look.**
   Evidence: compare implemented states/variants against the step-4 anatomy doc.
4. **(From Result/DoD) Output was visually verified (screenshot or live render) against
   the Stitch project/anatomy source.**
   Evidence: render proof distinct from source code review alone.

See `agent_doc/README.md` for the verdict scale and reporting format.
